    REDIS

    A NEW KEEPER OF MEMORY

    水晶不是真本。
    它只是记住答案。

    档案员没有烧掉《百职录》，也没有把真相交给水晶。他只改了来访的次序：

    1. 01
      先问水晶GET archive:profession:{id}
    2. 02
      水晶忘了，才翻真本CACHE MISS → SELECT MYSQL
    3. 03
      把答案留在水晶里SET + TTL 300s

    MySQL 仍是唯一真本。Redis 随时可以被清空；它的价值，不是替代真相，而是替真相挡住重复的追问。

    END OF CHAPTER I

    那天以后，《百职录》仍锁在长廊尽头。

    只是绝大多数来访者，再也不必惊动它。

    档案员终于可以坐下喝完一杯还热着的茶。水晶替他回答那些重复的问题；而当答案改变，他仍会先改写真本，再让旧记忆失效。

    缓存适合回答“它是什么”。
    却不能裁决——最后一个名额，究竟属于谁。
